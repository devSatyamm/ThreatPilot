// 03 · PIPELINE

The 5-Phase Attack Pipeline

01
PHASE 01 · INPUT

Target Submission

The user submits a target — either a web application URL or the path to a native mobile APK/IPA file. ThreatPilot validates the input, initializes an isolated Docker sandbox environment, and creates a scan session with a unique ID.

URL or APK/IPA accepted Docker sandbox isolation Session ID generated CI/CD webhook trigger
INPUT SCHEMA
target: "https://..."
mode: "full_attack"
modules: ["sqli","xss"]
intensity: "critical"
02
PHASE 02 · RECON

Autonomous Crawling

Playwright-powered headless browser crawls every accessible route. For mobile apps, Frida instruments the runtime to intercept API calls and map network traffic. The result is a complete attack surface inventory with endpoints, parameters, and authentication flows.

Playwright headless crawl Frida mobile instrumentation API route discovery Form & parameter mapping Auth flow traversal
RECON OUTPUT
endpoints: 47
forms: 12
parameters: 89
auth_flows: 3
03
PHASE 03 · ATTACK

AI Attack Engine

GPT-4o processes the attack surface map and generates a contextual attack strategy — not template payloads, but adaptive, target-specific vectors. The attack engine executes SQLi, XSS, auth bypass, brute force, CSRF, and traversal attacks across all mapped endpoints simultaneously.

GPT-4o strategy generation Adaptive payload mutation Parallel multi-vector attacks Rate-limited to avoid lockout LangChain agent orchestration
ATTACK STATS
vectors: 47
payloads_sent: 1,240
parallel_threads: 12
duration: 18s
04
PHASE 04 · ANALYSIS

AI Response Analysis

Every HTTP response, error message, and behavioral anomaly is fed back to the LLM for classification. The AI distinguishes true vulnerabilities from noise, assigns CVSS v3.1 severity scores, and identifies exploit chains — correlating multiple vulnerabilities that can be combined for greater impact.

LLM response classification False positive elimination CVSS v3.1 scoring Exploit chain correlation Confidence scoring
ANALYSIS OUTPUT
confirmed: 5
false_pos: 3
chains: 2
avg_cvss: 8.4
05
PHASE 05 · REPORT

Report Generation

A comprehensive security report is auto-generated with CVE-style IDs, reproduction steps, affected endpoints, CVSS scores, and AI-suggested code fixes. Delivered as JSON for CI/CD pipelines, PDF for executives, or directly to Jira/Slack for dev teams — in seconds.

CVE-style IDs assigned AI code fix suggestions JSON / PDF / Webhook export Jira ticket auto-creation Slack alert delivery
REPORT METRICS

medium: 1
total_time: ~30s
